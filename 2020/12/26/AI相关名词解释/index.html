Beam Search

在从w1得到w2的过程中，一般是得到了所有词表中单词出现的概率的排序，w2是出现概率最大单词（一般是用softmax回归得到概率），Beam Search如果把Beam设置为n，算法每次搜索到概率最大的n个单词，而不是一个单词。

假如词表大小为5，里面有abcde五个单词，Beam大小为2，预测到的第一个单词是c和e，单词是c时下个输出可能是ca,cb,cc,cd,ce，单词是e时，下个输出是ea,eb,ec,ed,ee，我们在这10个可能中挑到概率最大的两个组合ce和eb，依次再以e和b为输入获得10个组合可能，找到概率最大的两个组合，这就是Beam Search的基本原理。

Tokenization

标识化

我们一般所说的标识化就是将整句分割为单个标识符（tokens）。

Batch

  1. 定义

    Batch：批处理，顾名思义就是对某对象进行批量的处理。

    ​ 训练神经网络时，在数据集很大的情况下，不能一次性载入全部的数据进行训练，电脑会支撑不住，其次全样本训练对于非凸损失函数会出现局部最优，所以要将大的数据集分割进行分批处理。
    batch_size就是每批处理的样本的个数。

    ​ 备注：

    • 对于凸函数而言，最优解一定是全局最优解；而对于非凸函数而言，最优解可能是局部最优解。
    • 梯度下降法是最常见的优化算法，在最小化损失函数时，通过梯度下降法一步步的迭代求解，得到最小化的损失函数和模型参数值。

  2. 如何设置batch_size?

    • 过大的batch_size会降低梯度下降的随机性，以至于更稳定，训练更容易收敛，但因此更容易陷入局部最优点

    • 较小的批次内存利用率极低，还会带来幅度和随机性较大的权重更新，比较容易震荡导致不稳定，但也因此能够跳出局部最小。

Epoch

  1. 定义

    ​ epoch是所有训练数据的训练次数。一个完整的数据集通过神经网络一次并返回了一次的过程(正向传递+反向传递)称为一个epoch。
    备注：
    ​ 在神经网络中传递完整的数据集一次是不够的，即使用一次迭代过程更新权重一次不够，epoch的数目需要>1，epoch数量增加，神经网络中的权重更新次数增加，曲线从欠拟合变为过拟合。

  2. 公式

    单次epoch=（全部训练样本/batch_size） / iteration =1

    iterations=number of batches=总样本数/batch_size

    即迭代的次数等于batch的数目

Mask

NLP中的mask

Representation

文本表示

将非结构化信息转化为结构化信息，这样就可以针对文本信息做计算，来完成日常所见的文本分类和情感判断任务

分类

  • one-hot编码 | 独热编码
    • 无法表示词语之间的关系
    • 过于稀疏的向量，导致计算和存储的效率不高
  • 整数编码
    • 同样无法表达词语之间的关系
    • 对于模型解释，整数编码可能具有挑战性
  • 词嵌入 | word embedding
    • 将文本通过一个低维向量表达
    • 语意相似的词在向量空间比较近
    • 通用性强
Word Embedding

词嵌入

2种主流算法

Word2vec：基于统计方法来获得词向量的方法，在2018年前主流，随着BERT和GPT2.0的出现，这种方式效果算不上最好。所以上一代的产物，现在已经不再使用Word Embedding

训练模式

  • 通过上下文来预测当前词：CBOW-(Continuous Bag-of-Words Model)

    一句话扣掉一个词，让你猜这个词

  • 通过当前词来预测上下文：Skip-gram(Continuous Skip-gram Model)

    给你一个词，让你猜前面和后面可能出现什么词

优缺点

  • 优点
    • 会考虑上下文，比之前的Embedding效果好
    • 维度更少，速度更快
    • 通用性强
  • 缺点
    • 词和向量是一对一的关系，无法解决多义词问题
    • 是一种静态的方式，无法针对特定任务做动态优化

GloVe：对前一种的扩展，将全局统计和基于上下文学习结合

BERT

Bidirectional Encoder Representation from Transformers

Encoder-Decoder

不特指某种算法，而是一类算法的统称，是一种通用框架

Encoder 称作编码器：将现实问题转化为数学问题

Decoder 称作解码器：求解数学问题，并转化为现实世界的解决方案

说明：

  • 连接Encoder和Decoder的向量长度固定

  • 根据不同任务可以选择不同的编码器和解码器

  • 存在的问题是当输入信息太长，会丢失掉一些信息

    • attention机制可以解决这个问题
Seq2Seq

Sequence-to-sequence 的缩写：输入一个序列，输出一个序列

最重要的地方是输入序列和输出序列长度可变

Seq2Seq属于Encoder-Decoder，

满足[输入序列，输出序列]的都可统称为Seq2Seq

LSTM

Long short-term memory 的缩写：一种特殊的RNN

Transformer

Transformer是一个完全基于注意力机制的编解码器模型，抛弃了循环与卷积结构，而采用自注意力机制

集束搜索和贪心解码
