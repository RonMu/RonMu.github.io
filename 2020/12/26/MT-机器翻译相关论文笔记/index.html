Attention is All You Need

Transformer笔记

模型

模型看成是黑箱，seq2seq，机器翻译中就是输入一种语言，输出另一种语言。

黑箱内，由编码组件encoders、解码组件decoders和它们之间的连接组成。

编码组件由编码器Encoder构成，多个编码器叠在一起；解码组件是由与编码器相同数量的解码器Decoder组成。

编码器在结构上是相同的，但是参数不共享。每个解码器分解为两个子层，首先句子会经过自注意力层self-attention（帮助编码器在对每个单词编码时关注输入句子的其他单词）。

自注意力层的输出传递到前馈神经网络（feed-forward）。

解码器除了这两层，在之间还有一个注意力层。

张量

张量是矢量概念的推广，矢量是一阶张量，矩阵是二阶张量。

输入单词通过词嵌入算法转换为词向量。词嵌入只发生在最底层的编码器中，最底层编码器中接受的向量列表即词向量，而其他编码器接收的是低一层编码器的输出（一个向量列表），向量列表大小是我们可以设置的超参数，一般是训练集中最长句子的长度。

输入序列中每一个词进行词嵌入后都将流经编码器的两个子层。

Transformer的一个特性：在这里输入序列中每个单词都有自己独特路径流经编码器，在self-attention中路径存在依赖关系，feed-forward层没有这些依赖关系，所以后者可以并行操作。

self-attention机制

模型处理输入序列的每个单词，自注意力机制会关注输入序列的所有单词，帮助模型对本单词更好地编码。

计算自注意力：1.从每个编码器输入向量中生成三个向量（查询向量、键向量、值向量，由词嵌入与三个权重矩阵相乘创建的），这些新向量在维度上可以比词嵌入向量更低。2.计算得分，拿输入句子的每个单词对当前处理词向量打分，这些分代表在编码当前单词时多重视句子的其他部分。分数计算方式是通过打分单词（所有输入句子的单词）的键向量与当前单词的查询向量点积得到的。3.将分数除以$\sqrt d_k$，由于论文使用的键向量维数为64，则该值为8。4.通过softmax传递结果，将所有单词分数归一化，得到的分数都是正值且和为1（softmax分数决定每个单词对编码当下位置的贡献，已经在这个位置的单词将获得最高的softmax分数）。5.将每个值向量乘以softmax分数（为了准备之后将它们求和），关注语义上相关的单词即分数高的。6.对加权值向量求和，即得到当前词向量在自注意力层的输出。（简单小结：编码某个单词时将所有单词值向量加权求和，而权重就是键向量与查询向量的点积再通过softmax得到）

公式：$Z=softmax(\frac{Q\times K^T }{\sqrt d_k})V$

Z为输出矩阵，Q为查询矩阵，K为键矩阵，V为值矩阵，$d_k$为键向量维数

multi-headed attention机制

多做几次同样的事情（Q、K、V通过参数矩阵映射），然后把结果拼接。

1.扩展模型专注于不同位置的能力。

2.给出注意力层的多个“表示子空间”。

Position Embedding 位置编码

位置编码表示序列的顺序。

$PE_{2i}(p)=\sin(\frac{p}{10000}^{\frac{2i}{d_{pos}}})$

$PE_{2i+1}(p)=\cos(\frac{p}{10000}^{\frac{2i}{d_{pos}}})$

如果没有这个，那么将K、V按行打乱顺序，Attention的结果还是一样的。

本身这个是绝对位置信息，而通过$\sin(\alpha+\beta)=\sin\alpha\cos\beta+\cos\alpha\sin\beta和\cos(\alpha+\beta)=\cos\alpha\cos\beta-\sin\alpha\sin\beta$，使$p+k$向量可以表示成$p$的向量的线性变换，可能用于表示相对位置。

位置编码向量可以帮助确定每个单词的位置或序列中不同单词之间的距离。

解码器后的线性变换和softmax

线性变换层将浮点数变成单词。

是一个简单的全连接神经网络，把解码组件产生的向量投射到一个比它大得多的、被称作对数几率的向量（这个向量就是输出词表的英语单词数量对应长度的向量，每一个单元格对应单词的分数）里。

softmax将分数变成概率（正数，上限1），概率最高的单元格被选中，并且对应单词作为输出。

attention机制的粗略理解

从关注全部到关注重点

将有限的注意力集中在重点信息上，从而节省资源，快速获得最有效的信息

参数少、速度快、效果好

  • 参数少

    模型复杂度小于CNN、RNN，参数少，对算力要求更小

  • 速度快

    RNN不能并行运算，attention每一步计算不依赖上一步结果，可以和CNN一样并行处理

  • 效果好

    attention之前长距离信息会被弱化

    attention是挑重点，就算文本比较长，也能从中间抓住重点，不丢失重要的信息

寻址过程

给定查询向量，通过计算与键向量的注意力分布附加在值向量上，求出attention值

句子分词$\rightarrow$词向量$x\in R^{1\times d}$，即$d$维行向量

句子$\rightarrow$矩阵$X\in R^{n\times d}$

编码序列：

1.RNN，$y_t=f(y_{t-1},x_t)$

​ 递归进行，无法并行，是马尔科夫决策过程

2.CNN，$y_t=f(x_{t-1},x_t,x_{t+1})$ 卷积尺寸为3

​ 方便并行，容易捕捉全局结构信息

3.Attention，$y_t=f(x_t,A,B)$

​ 获取全局信息

soft attention

  • key = value = x
  • key ！= value

Improving Grammatical Error Correction via Pre-Training a Copy-Augmented Architecture with Unlabeled Data

NMT成为GEC的SOTA，

本文通过将无变化单词从原句复制到目标句，提出一种复制增强的架构。

由于缺乏足够标注数据，来达到高精度。用十亿无标注基准数据来预训练带有去噪的自动编码器的复制增强架构，并且比较了完全训练与部分训练的预训练模型。

Introduction

GEC：检测并纠正语法错误

Example:

$Nothing\quad is\quad [\boldsymbol{absolute}\quad \rightarrow\quad absolutely]\quad right\quad or\quad wrong.$

即使机器翻译系统成为GEC最先进的方法，GEC还是有异于翻译，因为GEC仅改动源句的少数单词。(More than 80% of the words can be copied from the source sentence.)

由于上述原因，本文通过允许从源句复制未改变单词和超出词典的单词来加强当前神经网络架构。

复制机制曾经用于文本摘要任务，这是第一次用于GEC任务。

GEC中，复制机制由于能直接从输入源符号中直接复制未改变单词和超出词典单词，使得训练模型可以只用很少词汇量。

summary
  1. A more proper neural architecture for the GEC problem, which is based on copy mechannism.
  2. Pre-train the copy-augmented model with large-scale unlabeled data using de-noising auto-encoders.
  3. Evaluate the architecture on the CoNLL-2014 test set, and get SOTA.

Our Approach

Base Architecture

Baseline

Attention based ==transformer== architecture as baseline

$h^{src}{ 1\cdots N} = encoder(L^{src}x{1\cdots N})$

$h_t = decoder(L^{trg}y_{t−1\cdots 1}, h^{src}_{1\cdots N})$

$P_t(w) = softmax(L^{trg}h_t)$

Loss

$l_{ce}=-\sum\limits_{t=1}^Tlog(p_t(y_t))$

损失为解码时候每个位置的交叉熵损失的累计。

Copying Mechanism

Copy机制的模型结构图如下所示，其主要思想是在生成序列过程中，考虑两个生成分布：分别是输入序列中的词语概率分布以及词典中的词语概率分布，将两者的概率分布加权求和作为最终生成的概率分布，进而预测每一个时刻生成的词语。该方法可以有效地利用输入语句和生成语句之间重叠词语多的特性，将简单的词语复制任务交给了Copy机制，将模型结构中的Attention等结构更多地用来学习比较难的新词生成。

Probability Distribution

$p_t(w)=(1-\alpha_t^{copy})*p_t^{gen}(w)+(\alpha_t^{copy})*p_t^{copy}(w)$

复制与生成之间的平衡取决于每一时刻$t$的平衡因子$\alpha_t^{copy}\in [0, 1]$.

$\alpha_t^{copy}=sigmoid(W^T\sum(A^T_t\cdot V))$

Copy Attention

copy分值是通过attention分布来计算的，是关于当前隐层和输入层之间的attention分布。

$q_t,K,V=h_t^{trg}W_q^T,H^{src}W_k^T,H^{src}W^T_v$

$A_t=a^T_tK$

$P^{copy}_t(w)=softmax(A_t)$

上述三个方程与$encoder-decoder \quad attention$基本一致。

Pre-training

Denoising Auto-encoder

降噪自编码

破坏句子的方式：

  • 10%删除token
  • 10%增加token
  • 10%从词典中随机选择一个单词替换当前单词
  • 通过在单词的位置上增加正态分布偏差来对单词进行排序，然后将单词按照校正后的位置重新排序，标准差为0.5。（打散单词）
Pre-training Decoder

预训练解码

作者初始化拷贝机制的decoer通过预训练参数形式，其他参数进行随机初始化。

Multi-Task Learning

Explore two different tasks for GEC to improve the performance.

Token-level Labeling Task

token级别的标注任务

标识原始句子中的每个词语是否正确。

对于原始句子中的每个词语$x_i$，对应于目标句子中的一个词语$y_j$，如果$x_i=y_j$，则将其标注为正确，否则则为错误。预测的时候是通过将$encoder$最终状态$h^{src}_i$送入$softmax$。

$p(label_i|x_{1\cdots N})=softmax(W^Th^{src}_i)$

Sentence-level Copying Task

句子级别的拷贝任务

这个任务的主要目的是当句子看似整体正确时，做尽可能多的拷贝。训练过程中将相等数量的正确句子和修改的句子对输入到模型中，当输入的是正确句子时，移除decoder层的atttntion处理

BART: Denoising Sequence-to-Sequence Pre-training for Natural Language Generation, Translation, and Comprehension

Introduction

BART: Bidirectional and Auto-Regressive Transformers, a denoising autoencoder for pretraining sequence-to-sequence models.

GPT: auto-regressive, left-to-right decoder, 单向解码器

BERT: auto-encoding, bidirectional encoder, 双向编码器

BART在利用encoder端的双向建模能力的同时，保留decoder自回归特性。

其实就是输入破坏文本到encoder，然后在decoder中还原。

训练方式:

  1. corrupting text with an arbitrary noising function.
  2. learning a model to reconstruct the original text.

随机改组原句顺序并使用一种全新的填充方案（随机将一段连续token<称为span>替换为mask）

It matches the performance of RoBERTa with comparable training resources on GLUE and SQuAD, achieves new stateof-the-art results on a range of abstractive dialogue, question answering, and summarization tasks, with gains of up to 6 ==ROUGE==.

ROUGE

Recall-Oriented Understudy for Gisting Evaluation

ROUGE-N: Overlap of N-grams between the system(automatically produced summary or translation) and reference summaries (typically humanproduced).

在系统摘要与参考摘要之间比较的文本粒度。

6 ROUGE，即系统摘要与参考摘要之间的六字母组重叠。

Example:

system:'the cat was found under the bed'

reference:'the cat was under the bed'

Bigrams:

system:the cat, cat was, was found, found under, under the, the bed

reference:the cat, cat was, was under, under the, the bed

Explanation:

$Recall=\frac{number\quad of\quad overlapping\quad words}{total\quad words\quad in\quad reference\quad summary}=\frac 66=1.0$

This means that all the words in the reference summary has been captured by the system summary, which indeed is the case for this example

$Precision=\frac{number\quad of\quad overlapping\quad words}{total\quad words\quad in\quad system\quad summary}=\frac 67=0.86$

This simply means that 6 out of the 7 words in the system summary were in fact relevant or needed.

$ROUGE-2_{Recall}=\frac 45=0.8$

$ROUGE-2_{Precision}=\frac 46=0.67$

关于ROUGE-L, ROUGE-S可以参考ROUGE详细介绍

Model

BART is a denoising autoencoder that maps a corrupted document to the original document it was derived from. It is implemented as a sequence-to-sequence model with a bidirectional encoder over corrupted text and a left-to-right autoregressive decoder.

Architecture(架构)

seq2seq Transformer

将ReLU激活函数修改为GeLU

  • 基础模型

    • 6层encoder-decoder

  • 大型模型

    • 12层

架构与BERT架构密切相关，但是有以下区别：

  1. 解码器对编码器的隐藏层有交叉注意力
  2. BART没有feed-forward网络
  3. BART的参数比同等大小BERT多10%
Pre-training BART

预训练方式：

破坏原始文件，优化解码器输出与原始文件之间的交叉熵这类重建损失（损失函数）

BERT仅有一种noise

而BART有多种：

  • Token Masking：BERT采用的
  • Token Deletion：随机删去token
  • Text Infilling：随机将一段连续token即span进行[MASK]替换
  • Sentence Permutation：将文本中句子打乱
  • Document Rotation：随机选择一个token作为document的开头

Fine-tuning BART(微调)
  • 序列分类任务
    • 类似于BERT中的CLS token
  • 符号分类任务
    • 将完整文档输入到编码器和解码器中，使用解码器最上方的隐藏状态作为单词表征用于分类token
  • 序列生成任务
    • 自回归解码器可以针对序列生成任务进行微调
  • 机器翻译
    • 端到端的方式接受训练
