强大的find

  • find ./|wc -l查看当前目录下文件个数
  • find ./ -name "core*" | xargs file搜寻文件和目录
  • find ./ -name '*.o'查找目标文件夹是否有obj文件
  • find ./ -name "*.o" -exec rm {}\;删除所有.o文件（当前目录及其子目录）
  • find是实时查找，想要更快推荐locate，但需要updatedb来更新数据库以获得最新文建索引信息

文件与权限管理

文件与目录权限的表示方法

查看文件权限

-rw———-1 root root 1523 sep 28 12:05 anaconda-ks.cfg

- rw———-1 root root 1523 sep 28 12:05 anaconda-ks.cfg
类型 权限 所属用户和组 文件名
  • 类型
    • -：普通文件
    • d：目录文件
    • b：块特殊文件
      • 挂载硬盘
    • c：字符特殊文件
      • 终端
    • l：符号链接
      • 快捷方式
    • f：命令管道
    • s：套接字文件
  • 权限
    • r=4 w=2 x=1
    • 前三个字符是当前用户对该文件的权限，中间三个字符是用户组对该文件的权限，最后三个字符是除了当前组其他用户
    • r：读
    • w：写
    • x：执行
    • 上述是对普通文件的权限
    • 目录权限
      • x：进入目录
      • rx：显示目录内文件名
      • wx：修改目录内文件名，删除目录
  • 用户和组
修改权限

chmod

  • chmod u+x /tmp/testfile
    • u 用户 g 用户组 o 其他 a 全部
    • + 增加权限 - 减少权限 = 设置权限
  • chmod 755 /tmp/testfile
    • 三个数字分别对应ugo
    • 7=1+2+4 所以rwx均有
    • 5=1+4 所以没有写的权限
    • 创建文件时默认权限644（666-022）022是umask值

chown更改属主属组

  • chown user /test将test目录的属主更改为user
  • chown :group /test将test目录的属组更改为group

chgrp仅更改属组

chgrp group /test注意没有冒号

注意如果属主和属组权限矛盾以属主为准

特殊权限

SUID 用于二进制可执行文件，执行命令时取得文件属主权限

  • chmod 4xxx file
  • xxx为原权限

SGID 用于目录，在该目录下创建新的文件和目录，权限自动更改为该目录的属组

SBIT 用于目录，该目录下新建的文件和目录，仅root和自己可以删除

  • /tmp
  • chmod 1xxx file

文件系统管理

常见文件系统

ext4 xfs NTFS（需安装额外软件ntfs-3g）

  • ext4
    • 超级块 文件总数 df查看 硬盘被格式化恢复就是恢复超级块
    • inode 记录每一个文件的名称、大小、权限 ls统计i节点
    • 数据块（datablock） 记录数据，一个接一个的链接式结构 du统计数据块大小
i节点和数据块操作

一个数据块大小默认4k，存的文件小于4k依然会是4k

mv改名不会改变i节点和block

rm 其实就是断开文件名与i节点

文件恢复就是在断开i节点之后马上断电，再用专业工具扫描盘符

ln 增加（硬）链接，可以使两个文件有相同i节点，这项操作不能跨越文件系统，也就是到了其他分区链接不存在

ln -s 增加软（符号）链接，i节点不同，注意对符号链接文件的权限设置会作用于原文件

facl扩充了属主、属组、其他用户三个位置不够的情况

chmod u+rx 1.sh

!/bin/bash 写在脚本开头，bash执行时会当作注释，而其他shell不会

bash 1.sh 不用赋予可执行权限

./1.sh 必须有可执行权限

source ./1.sh 如果有更换目录的命令，当前目录也会发生变化

. ./1.sh 同上，对当前环境造成影响

原因是前两种会创建子进程，运行完回到父进程

内建命令不产生子进程，对当前shell生效

管道和重定向

管道和信号一样是进程通信方式

连接两个程序，把第一个程序输出作为第二个程序输入

管道符|

由于管道本质上建立了子进程，所以不推荐在其中写入内建命令，不会传递给父进程，父进程无作用

一个>会清空，两个>会追加，2>输出错误，&>全部重定向无论正确错误

wc -l 统计输入的行数

1.txt << EOF 输入EOF才结束输入

解释型语言，不建议用来计算

变量值中若有空格放在” “或单引号，如string=hello bash后面的bash会被认为是命令，所以用string=”hello bash”

变量有默认作用范围，不在子进程和平行进程中生效

export varname 导出变量，子进程也可以得到父进程的变量

unset varname 删除变量

自己添加路径环境变量，就可以把自己写的shell脚本变成命令，环境变量会自动导出，子进程可以继承

环境变量配置文件

non-login shell 两个profile不加载，配置文件加载不完全

login shell 也就是 su - username 才有四个配置文件加载

su 是不改变当前环境变量

su - 是改变为切换到用户的环境变量

也就是说su只能获得username的执行权限，不能获得环境变量

而su -是切换到username并获得username的环境变量及执行权限

/etc/profile
当使用su - root，这个文件第一个被加载

终端启动的时候的环境
不要在/etc/profile里面直接修改，而是创建一个custom.sh在/etc/profile.d里面制作自定义的修改
